2. Objet de cette politique

Cette politique explique quelles donnees sont traitees dans ScanDoc, pourquoi, comment, avec qui elles peuvent etre partagees, et quels sont vos droits.

ScanDoc est une application mobile de scan, edition, OCR, export PDF/image et partage de documents.

3. Principe general de ScanDoc

Par defaut, ScanDoc fonctionne en local sur votre appareil.

  • Pas de compte obligatoire en V1 publique.
  • Pas de synchronisation cloud publique active en V1 publique.
  • Pas d'abonnement in-app actif en V1 publique.
  • Pas de permission de stockage globale necessaire pour importer depuis galerie/fichiers (pickers systeme).
  • Le module cloud (Supabase) n'est utilise que si une configuration runtime existe et si l'utilisateur declenche explicitement un flux cloud (connexion, sauvegarde, restauration).

4. Donnees traitees

4.1 Donnees que vous creez dans l'app

  • Photos/documents scannes (images d'origine et traitees)
  • PDF generes
  • Miniatures de documents
  • Texte OCR extrait
  • Signatures enregistrees
  • Textes predefinis enregistres
  • Noms de documents et dossiers

4.2 Donnees techniques et preferences

  • Langue choisie
  • Preferences d'affichage et de scan
  • Etats d'acces premium/recompenses publicitaires
  • Compteurs de certaines actions (ex. regles de diffusion pub/rewarded)
  • Etat local de session cloud si fonctionnalite activee en environnement compatible
  • Si cloud active : e-mail de compte cloud, statut de connexion, horodatages de derniere sauvegarde/restauration et dernier message d'erreur cloud

4.3 Donnees publicitaires (Google AdMob)

La V1 gratuite affiche des annonces via Google AdMob. Dans ce cadre, Google peut traiter, selon ses propres regles :

  • Identifiants publicitaires et techniques (ex. Ad ID, identifiants d'instance SDK)
  • Informations appareil/reseau (ex. modele appareil, version OS, langue, adresse IP approximative, operateur, type de reseau)
  • Donnees de diffusion/interaction publicitaire (impressions, clics, ouvertures, diagnostics de diffusion)
  • Donnees de mesure et de lutte contre la fraude publicitaire

Implementation actuelle de l'app :

  • ScanDoc demande des annonces non personnalisees quand le SDK le permet (requestNonPersonalizedAdsOnly).
  • Les parametres de marquage enfant/consentement mineur sont configures cote SDK AdMob, mais les traitements publicitaires restent regis par Google.
  • Regles de confidentialite Google
  • Fonctionnement des annonces Google

5. Permissions Android

ScanDoc applique le minimum necessaire.

  • Camera (android.permission.CAMERA) : utilisee pour la capture de documents.
  • Import galerie/fichiers : via pickers systeme (expo-image-picker, expo-document-picker), sans permission media globale.
  • Permissions explicitement bloquees : audio et stockage large non necessaires (RECORD_AUDIO, READ/WRITE_EXTERNAL_STORAGE, READ_MEDIA_*).

6. Finalites et bases legales (RGPD)

  • Execution du service demande : scan, recadrage, edition, OCR, export et partage.
  • Consentement / action utilisateur : autorisation camera, partage vers app tierce, impression.
  • Interet legitime : securite, prevention d'abus, stabilite technique, monetisation publicitaire en version gratuite.
  • Obligations legales : le cas echeant, respect des obligations applicables.

7. Partage des donnees

  1. Vers le service/app choisi explicitement par l'utilisateur (ex. Telegram, WhatsApp, e-mail, impression).
  2. Vers Google AdMob pour la diffusion et la mesure publicitaire.
  3. Vers Supabase uniquement si le cloud est activement utilise (connexion, sauvegarde, restauration declenchees par l'utilisateur).

Hors de ces cas, ScanDoc ne vend pas vos donnees.

  • Privacy Policy Supabase
  • DPA / securite Supabase

8. Hebergement et transferts hors UE

  • Les donnees locales restent sur votre appareil.
  • Les services tiers (AdMob, eventuellement Supabase si active) peuvent impliquer des traitements hors UE selon leurs infrastructures.
  • Ces transferts sont encadres par les conditions et garanties de ces fournisseurs.

9. Duree de conservation

  • Local (appareil) : jusqu'a suppression par l'utilisateur ou desinstallation.
  • Partage externe : selon le service destinataire choisi.
  • Publicite (AdMob) : selon la politique de conservation de Google.
  • Cloud (si active) : tant que la sauvegarde existe sur le compte/service concerne.

10. Securite

Des mesures techniques raisonnables sont appliquees :

  • Stockage local dans l'espace applicatif (sandbox) du systeme mobile
  • Minimisation des permissions Android
  • Partage/export uniquement via des actions explicites utilisateur
  • Controles applicatifs sur operations sensibles (ex. verification d'existence de fichier avant partage, prevention de doubles demandes concurrentes)
  • Pour Supabase (si active) : acces authentifie, URL signees a duree courte pour restauration, canaux reseau securises

ScanDoc n'active pas de SDK dedie d'analytics/crash-reporting dans le parcours V1 public.

Aucun systeme n'est totalement exempt de risque. L'utilisateur reste responsable de la protection de son appareil.
